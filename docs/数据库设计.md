**接口自动化测试平台 数据库设计文档（V1）**

---

**一、通用设计约定**

- 数据库名：`slow_platform`（示例名，可自行调整）  
- 字符集：`utf8mb4`  
- 排序规则：`utf8mb4_general_ci`  
- 主键：统一使用 `BIGINT UNSIGNED` 自增字段 `id` 作为主键  
- 时间字段约定：  
  - `created_at`：记录创建时间，`DATETIME`  
  - `updated_at`：记录最近更新时间，`DATETIME`  
- 布尔值：统一使用 `TINYINT(1)`，0 表示否 / false，1 表示是 / true  
- JSON：使用 MySQL 原生 `JSON` 类型（建议 MySQL 5.7+）

#### 2.6.2 `system_config` —— 系统配置表

### 2.7 索引与数据量考虑（补充）

- 所有外键字段（`project_id`、`environment_id`、`scene_id` 等）都建议建立普通索引，以提升列表过滤和关联查询性能。  
- `execution` 与 `execution_step` 会随时间增长较快，后续可按项目或时间做归档策略。  
- JSON 字段主要用于存储配置与快照，不推荐频繁做复杂 JSON 条件查询；若后续确有需要，可冗余关键字段为独立列。
